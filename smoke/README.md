# Sarf Atlas Smoke Prototype

This is a local prototype smoke-test area. Its purpose is to verify that Sarf
Atlas can drive Ember research runs without owning inference code.

Sarf Atlas does not implement hidden-state extraction. It should provide
research configs, prompts, and manifests, then invoke Ember CLI and backend
paths. Ember remains responsible for backend dispatch, artifact writing, and
run validation.

Real model paths and llama.cpp helper paths are user-local. Do not commit real
machine-specific paths into these example files.

## Expected Flow

1. Copy or edit `llama_cpp_smoke.example.toml`.
2. Point `model_path` to a local GGUF model.
3. Point `llama_cpp_binary` to a local patched/custom `llama-ember-extract`
   helper if using `llama-cpp-external`.
4. Run Ember extraction or use the helper script to print the command first.
5. Validate the produced run directory with Ember's backend comparison path or
   artifact-contract validation code.

Current Ember flow:

```text
sarf-atlas config/manifest
  -> Ember CLI
  -> Ember backend abstraction
  -> llama.cpp external backend
  -> Ember run artifacts
  -> Ember validate-backends
  -> gguf-parity-tools optional validation
```

## Ember CLI Shape

Ember has an `extract` subcommand:

```bash
cargo run --release -- extract --config sarf-atlas/smoke/llama_cpp_smoke.example.toml
```

The `llama-cpp-external` backend requires:

- an executable `llama_cpp_binary`
- an existing GGUF `model_path`
- an existing input JSONL file
- `layers = []` for now

Hidden-state layer extraction through `llama-cpp-external` is not wired yet.
The current smoke workflow is tokenization-only: no generation, no logits, and
no hidden states. Ember validates the produced artifact contract after the
external process exits.

Because the example config contains placeholder paths, it is documentation and
configuration scaffolding only. It is not expected to run successfully until
the local paths are replaced.

## Mock External Backend Smoke

`llama_cpp_mock_smoke.toml` exercises Ember's real `extract` CLI and
`llama-cpp-external` dispatch without running llama.cpp or loading a real model.
It points to `mock_llama_ember_extract.py`, which writes only the minimal
tokenization/position artifact skeleton that Ember validates today.

The mock extractor does not perform real inference, real llama.cpp
tokenization, logits extraction, or hidden-state extraction. Its outputs are
for backend plumbing validation only and must not be used as research results,
paper evidence, numerical evidence, tokenization evidence, or model behavior
evidence.

A successful mock smoke proves only that Ember can dispatch to an external
backend process and validate the returned artifact shape. A real llama.cpp smoke
test still requires a real patched/custom helper binary, a local GGUF model, and
a non-mock config with user-local paths.

Run it with:

```bash
cargo run -- extract --config sarf-atlas/smoke/llama_cpp_mock_smoke.toml
```

Expected output goes under `/tmp/sarf-atlas-ember-smoke/runs/`. This is a
backend wiring smoke only, not a numerical, tokenization, logits, or
hidden-state validation.

## Real Qwen3 llama.cpp Tokenization Smoke

The local tokenization smoke uses a private `*.local.toml` config or the
temporary config generated by `run_tokenization_parity_smoke.sh`. It points to a
local GGUF model and a real llama.cpp `llama-tokenize` binary through Ember's
`llama-cpp-external` dispatch. It validates that Sarf Atlas can ask Ember to
spawn an external llama.cpp-backed helper and receive artifact-contract-shaped
outputs for the smoke prompts.

This smoke performs real GGUF-backed tokenization only. It does not generate
text, compute logits, or extract hidden states, and its outputs are not research
results.

Build llama.cpp's tokenizer outside the repo, then run the guarded local flow:

```bash
MODEL_PATH=/path/to/model.gguf \
TOKENIZER_JSON=/path/to/tokenizer.json \
LLAMA_TOKENIZE_BIN=/path/to/llama-tokenize \
GGUF_PARITY_TOOLS_PATH=/path/to/gguf-parity-tools \
OUT_ROOT=/tmp/sarf-atlas-ember-smoke \
bash sarf-atlas/smoke/run_tokenization_parity_smoke.sh
```

Expected output goes under
`/tmp/sarf-atlas-ember-smoke/runs/tokenization-parity-smoke/`.

The helper also writes `metadata.llamacpp.json` in the run directory using the
`gguf-parity-tools token-audit` metadata shape.

To compare llama.cpp GGUF tokenization against the local tokenizer JSON:

```bash
python3 sarf-atlas/smoke/export_tokenizer_json_metadata.py \
  --tokenizer /path/to/tokenizer.json \
  --prompts sarf-atlas/smoke/prompts.small.jsonl \
  --out /tmp/sarf-atlas-ember-smoke/runs/tokenizer-json/metadata.tokenizer-json.json

PYTHONPATH=/path/to/gguf-parity-tools \
  python3 -m parity_tools token-audit \
  --candidate-metadata /tmp/sarf-atlas-ember-smoke/runs/tokenizer-json/metadata.tokenizer-json.json \
  --reference-metadata /tmp/sarf-atlas-ember-smoke/runs/tokenization-parity-smoke/metadata.llamacpp.json \
  --out /tmp/sarf-atlas-ember-smoke/runs/tokenization-parity-smoke/token-audit.json
```

## Optional Dry-Run Helper

To print the command without running inference:

```bash
bash sarf-atlas/smoke/run_smoke_dry_run.sh
```

To print a command for a copied local config:

```bash
bash sarf-atlas/smoke/run_smoke_dry_run.sh path/to/local-smoke.toml
```
